KRONOS SWARM CORE — Context File
====================================
Last updated: 2026-06-25

OVERVIEW
--------
A simulated AI-driven "swarm intelligence" engine for football match analysis.
Five distinct AI agents analyze in-game telemetry, produce textual verdicts,
and a fracture calculator measures swarm disagreement. Full-stack app:
Python SSE backend + React/TypeScript/Vite/Tailwind frontend.

CURRENT STATE — COMPLETE
------------------------
Backend: 119 tests passing, 110 subtests. Full SSE server on port 3000.
Frontend: Production-quality React app with 5 routes (Landing / Command Center /
Swarm Intelligence / Debate Transcript / Granite Intelligence Center),
mission-control aesthetic. SSE connection status indicator in header.

ARCHITECTURE / DATA FLOW
------------------------
SSE → KronosProvider → normalizeKronosPacket → KronosContext → useKronos() → Components

process_next_tick():
  → KronosMatchTicker.generate_tick() → KronosTelemetryPacket
  → Each of 5 agents constructs prompt → LLMGateway.generate() → LLMResponse
    → MockProvider uses PersonaBuilder (telemetry-aware templates, per-agent voice)
  → SwarmFractureCalculator.calculate(verdicts) → metrics
  → HeuristicValidator.validate(assessments, fracture) → ValidateOutput
  → GraniteReviewEngine.review(assessments, fracture, validation) → GraniteReview
  → Return { telemetry, debate_outputs, swarm_metrics, validation, granite_review }

Escalation thresholds (granite_review.py):
  fracture_index >= 75
  overall_confidence <= 0.30
  contradiction_count >= 5

LLM Gateway Modes:
  mock:   Always MockProvider
  bob:    Always BobProvider
  hybrid: Try BobProvider → MockProvider fallback
  granite: Always GraniteProvider

KEY DECISIONS
-------------
- React Router with 5 routes: / (landing), /war-room, /swarm, /transcript, /granite.
- Granite Intelligence Center (Phase 4.3): 6-stage Decision Trace (OBSERVE → ANALYZE → DEBATE → VALIDATE → GRANITE REVIEW → RECOMMENDATION) as explainability centerpiece.
- SSE connection status (CONNECTED/CONNECTING/OFFLINE) tracked in KronosProvider via EventSource lifecycle events, displayed in CommandHeader.
- Confidence classification labels (HIGH/STRONG/MODERATE/LOW/CRITICAL UNCERTAINTY) applied where percentages are shown.
- MockProvider generates personality-rich text via PersonaBuilder — telemetry-aware templates with per-agent voice, signal detection, and deterministic template selection.
- Frontend components read from single source of truth: useKronos() → KronosContext.
- Pipeline labels: SWARM VALIDATION LAYER, IBM GRANITE REVIEW LAYER, FINAL DECISION LAYER.
- GraniteReviewEngine uses named constants for escalation thresholds.
- Granite unavailable state uses graceful-degradation messaging (not failure language) across all UI locations.
- KronosDebugPanel removed from War Room for production demo (file retained for development).
